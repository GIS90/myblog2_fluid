Hexo基础命令学习

1、简介

昨天写了blog的初始化搭建(hexo + next)，乘胜追击，学习查看了一些hexo基础的命令，总结一下分享。

基于官方的手册，很基础，加了一些自己的见解。

2、预览

id name brief frequency
1 init 初始化 🌟
2 new 新建文章 🌟🌟🌟
3 generate 生产静态文件 🌟🌟🌟
4 publish 草稿发布 🌟

3、正文

3.1、init

新建一个网站。folder文件夹已存在，会重新建里一个新的blog，主要用户初始化。

3.2、new

新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。
一般情况有3种，page、post、draft

3.3、generate

生成静态文件，用于打包，第二行是简写。
选项 描述
-d, –deploy 文件生成后立即部署网站
-w, –watch 监视文件变动

3.4、publish

发表草稿，就是通过hexo new draft建立的文章。

3.5、server

启动服务器。默认情况下，访问网址为： http://127.0.0.1:4000/。
选项 描述
-p, –port 重设端口
-s, –static 只使用静态文件
-l, –log 启动日记记录，使用覆盖记录格式

3.6、deploy

部署网站，第二行简写。

参数 描述
-g, –generate 部署之前预先生成静态文件

3.7、render

渲染文件。

参数 描述
-o, –output 设置输出路径

3.8、migrate

从其他博客系统 迁移内容，还没用过。

3.9、clean

清除缓存文件 (db.json) 和已生成的静态文件 (public)。
在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

3.10、list

列出网站资料，types：page，post，route，tag，category。

3.11、version

显示 Hexo 版本。

4、选项

4.1、安全模式

在安全模式下，不会载入插件和脚本。当您在安装新插件遭遇问题时，可以尝试以安全模式重新执行。

4.2、调试模式

在终端中显示调试信息并记录到 debug.log。当您碰到问题时，可以尝试用调试模式重新执行一次，并提交调试信息到 GitHub。

4.3、简洁模式

隐藏终端信息。

4.4、自定义配置文件的路径

自定义配置文件的路径，执行后将不再使用 _config.yml。

4.5、显示草稿

显示 source/_drafts 文件夹中的草稿文章。

4.6、自定义 CWD

自定义当前工作目录（Current working directory）的路径。

5、个人建议

5.1、命令一

以-p参数指定服务的端口启动，加上–debug模式，便与调试
–draft 可以把草稿的文章也加载显示

5.2、命令二

重新生产静态文件并上传github.io，刷新github的网站。
